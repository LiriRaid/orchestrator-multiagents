# OpenSpec Flow

Use this flow for durable changes:

1. **Explore**
   - Understand existing code and constraints.
2. **Proposal**
   - Define the user goal, motivation, scope, and non-goals.
3. **Spec**
   - Describe required behavior and acceptance criteria.
4. **Design**
   - Document technical approach and tradeoffs.
5. **Tasks**
   - Break the work into executable pieces.
6. **Queue**
   - Convert ready tasks into `QUEUE.md`.
7. **Apply**
   - Let worker agents implement queued tasks.
8. **Verify**
   - Check implementation against proposal, spec, design, and tasks.
9. **Archive**
   - Close the change with a concise summary.

Small direct changes do not need the full flow.
